GHOST Protocol

Guarded Healthcare Output with Safe Transmission — PHI-Safe AI Assistant

🎯 Problem Statement

Healthcare teams increasingly want to use GenAI for clinical summarization, letter drafting, and lab analysis. However, uncontrolled use exposes PHI to external AI systems — creating HIPAA violations, compliance risk, and patient trust erosion. Many organizations respond by banning AI entirely, losing productivity gains.

The challenge: Not what AI can do — but how to use AI safely and responsibly with sensitive healthcare data.

💡 Our Solution

A Redact-then-Reason pattern: PHI is automatically detected and redacted in the browser before any data reaches an AI model. The solution enforces a strict 8-step pipeline with three mandatory human review checkpoints and a complete audit trail.

🔑 The pattern is the product — a repeatable, auditable framework for safe GenAI use in any regulated industry.

👤 Three human checkpoints enforce accountability

🧠 AI Approach

  • PHI Detection: Browser-side regex engine detecting 7+ HIPAA identifier types across multiple languages — one active at a time — plus user-defined custom rules with live regex testing
  • Redaction: Category-labeled token replacement (e.g., [PATIENT NAME REDACTED]) — preserves clinical context
  • LLM: Azure API with mode-specific system prompts (Summarize / Draft / Analyze) instructed to preserve redaction tokens
  • Risk Scoring: HIPAA-weighted compliance risk score (0–100) based on sensitivity of detected PHI types (SSN: 28, MRN: 20, DOB: 16)
  • Batch Mode: File-upload batch processing (.txt or .pdf) with per-file risk triage, AI summaries, and CSV export

👤 Human-in-the-Loop

Three explicit human checkpoints — architecturally enforced, not optional:

  • Checkpoint 1: User reviews detected PHI findings after scan — verifies what was found before redaction is applied
  • Checkpoint 2: User approves sanitized text before it is sent to AI — can reject and revise
  • Checkpoint 3: User approves AI output before it can be exported or used clinically — can reject and regenerate

Every approval/rejection is timestamped in the audit trail.

Feature Set: 8-step pipeline, 3 AI modes, PHI detection across multiple languages (one active at a time), custom rules with live regex testing & color coding, HIPAA-weighted risk scoring, file-upload batch processing for .txt and .pdf with CSV export, auto PDD generation, PDF referral letters, email compose, full audit dashboard.
